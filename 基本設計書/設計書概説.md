# ST管理システム 实施计划

## Context

开发一个完整的企业管理系统（ST管理システム），包含社员管理、顾客管理、勤务管理、请求书/注文书管理四大模块。技术栈：React + Ant Design 前端，Spring Boot 后端，PostgreSQL 数据库。

## 2. 数据库设计 (PostgreSQL)

### 7张表:

**user** - 登录用户

| 列名 | 类型 | 说明 |
|------|------|------|
| id | BIGSERIAL PK | |
| username | VARCHAR(50) UNIQUE | 登录名 |
| password | VARCHAR(255) | 密码       |
| role | VARCHAR(20) | ADMIN/USER |
| create_time | TIMESTAMP | 创建时间 |
| update_time | TIMESTAMP | 更新时间 |

**employee** - 社员信息

| 列名 | 类型 | 说明 |
|------|------|------|
| id | BIGSERIAL PK | |
| employee_code | VARCHAR(20) UNIQUE | 社员编号 |
| name | VARCHAR(100) | 姓名 |
| email | VARCHAR(255) | 邮箱 |
| phone | VARCHAR(20) | 电话号码 |
| address | VARCHAR(500) | 住址 |
| department | VARCHAR(100) | 部门 |
| position | VARCHAR(100) | 职位 |
| bank_name | VARCHAR(200) | 银行名 |
| bank_branch | VARCHAR(200) | 支店名 |
| bank_account_type | VARCHAR(20) | 账户类型 |
| bank_account_number | VARCHAR(50) | 账号 |
| bank_account_holder | VARCHAR(100) | 账户名义 |
| attachment_path | VARCHAR(500) | 附件路径 |
| create_time | TIMESTAMP | 创建时间 |
| update_time | TIMESTAMP | 更新时间 |

**customer** - 顾客信息

| 列名 | 类型 | 说明 |
|------|------|------|
| id | BIGSERIAL PK | |
| customer_code | VARCHAR(20) UNIQUE | 顾客编号 |
| company_name | VARCHAR(200) | 公司名 |
| president_name | VARCHAR(100) | 社长名 |
| website | VARCHAR(500) | 公司网站URL |
| address | VARCHAR(500) | 公司地址 |
| contact_name | VARCHAR(100) | 联系人 |

**bank_account** - 银行口座（★MASTERテーブル，机密信息管理）

| 列名 | 类型 | 说明 |

| id | BIGSERIAL PK | 内部主键 |

| account_type | VARCHAR(20) | 普通/当座 |

| account_holder | VARCHAR(100) | 口座名義 |

**attendance** - 勤务记录

| 列名 | 类型 | 说明 |

| id | BIGSERIAL PK | |

| 列名 | 类型 | 说明 |

| year | INT | 年份 |
| month | INT | 月份 |

## 3. 后端 API 设计

| Method | Path | 说明 |
|--------|------|------|
| POST | /api/auth/login | 登录 |
| GET | /api/auth/me | 获取当前用户 |
| GET/POST | /api/employee | 列表/新增 |
| GET/PUT/DELETE | /api/employee/{id} | 详情/修改/删除 |
| POST | /api/employee/{id}/upload | 上传附件 |
| GET | /api/employee/{id}/download | 下载附件 |
| GET/POST | /api/customer | 列表/新增 |
| GET/PUT/DELETE | /api/customer/{id} | 详情/修改/删除 |
| GET/POST | /api/bank-accounts | 银行口座一覧/追加（主テーブル） |
| GET/PUT/DELETE | /api/bank-accounts/{id} | 银行口座詳細(含顧客業務情報)/修改/删除 |
| GET/POST | /api/attendance | 列表(按月/社员)/新增 |
| PUT/DELETE | /api/attendance/{id} | 修改/删除 |
| GET | /api/attendance/export | 导出CSV |
| GET/POST | /api/invoice | 列表(按月/顾客)/新增 |
| GET/PUT/DELETE | /api/invoice/{id} | 详情/修改/删除 |
| POST | /api/invoice/{id}/documents | 上传注文书 |
| GET | /api/invoice/{id}/documents/{docId}/download | 下载注文书 |
| DELETE | /api/invoice/{id}/documents/{docId} | 删除注文书 |

## 4. 前端页面设计

### 4.1 登录画面 (Login.jsx)
- 用户名 + 密码输入框
- 登录按钮
- 登录失败错误提示

### 4.2 主布局 (Layout.jsx)
- 左侧 Sidebar: Logo + 5个菜单项(社员管理/顾客管理/勤务管理/请求书注文书管理/银行口座管理)
- 右上: 当前用户名 + 退出按钮
- 右侧: 内容区(React Router Outlet)

### 4.3 社员管理 (Employee.jsx)
- 搜索栏: 社员编号/姓名搜索 + 新规登录按钮
- Table: 社员编号/姓名/部门/邮箱/电话/操作(编辑/删除)
- Modal表单: 个人信息 + 银行信息 + 文件上传
- 文件下载按钮

### 4.4 顾客管理 (Customer.jsx)
- Table: 顾客编号/公司名/社长名/营业担当/事务担当/Web网站/操作
- Modal表单(800px): 公司信息 + 营业担当(姓名/电话/Email) + 事务担当(姓名/电话/Email)
- 银行口座管理: 専用ページ（サイドバー「🔒 銀行口座管理」メニューから遷移）。顧客選択後に口座一覧を表示。口座番号はマスク表示、表示確認後30秒自動再マスク。追加/編集/削除、安全警告バナー

### 4.5 勤务管理 (Attendance.jsx)
- 筛选: 年/月选择器 + 社员下拉
- Table: 日期/社员名/工作时间/加班时间/状态/备注
- 新规/编辑/删除
- CSV导出按钮

### 4.6 请求书/注文书管理 (Invoice.jsx)
- 筛选: 年/月选择器 + 顾客下拉
- Table: 请求书编号/顾客名/年月/金额/状态/操作
- Modal: 请求书编辑(编号/金额/状态/备注)
- 注文书Tab: 文件列表 + 上传/下载/删除

### 4.7 银行口座管理 (BankAccount.jsx)
- 専用ページ: サイドバー「🔒 銀行口座管理」メニューから遷移（機密情報分離）
- 顾客选择器: ドロップダウンで顧客を選択
- Table: 口座一覧（口座番号はマスク表示「****4321」）
- 表示/隠す: 👁 ボタンで口座番号を表示（確認後30秒自動再マスク）
- 操作: 追加/編集/削除
- 安全警告バナー: 操作ログ記録の旨を表示

## 5. 实施步骤

### Step 1: 数据库初始化
- 创建 PostgreSQL 数据库 `st_management`
- 执行 `database/init.sql` 创建所有表
- 插入初始管理员用户

### Step 2: Spring Boot 后端搭建
- 创建 Spring Boot 项目(pom.xml)
- 配置 application.yml (DB连接, JWT密钥, 文件上传路径)
- 实现 Security + JWT 认证
- 实现各模块 Model → Repository → Service → Controller

### Step 3: React 前端搭建
- 使用 Vite 创建 React 项目
- 安装依赖: antd, axios, react-router-dom, i18next, react-i18next, dayjs
- 配置 i18next 日/中双语
- 实现 Login → Layout → 4个功能页面

### Step 4: 联调测试
- 启动后端(端口8080)
- 启动前端(端口3000)
- 验证所有CRUD操作和文件上传下载
