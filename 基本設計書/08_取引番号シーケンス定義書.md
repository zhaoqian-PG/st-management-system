## 2. torihiki_seq（取引番号シーケンス）

### 2.1 基本情報

| 項目 | 値 |
|------|-----|
| シーケンス名 | `torihiki_seq` |
| スキーマ | `public` |
| データ型 | BIGINT |
| 開始値 (START) | 1 |
| 増分 (INCREMENT) | 1 |
| 最小値 (MINVALUE) | 1 |
| 最大値 (MAXVALUE) | 999999 |
| キャッシュ (CACHE) | 1（デフォルト） |
| サイクル (CYCLE) | NO（上限到達時エラー） |
| 所有者 | bank_account テーブル |

### 2.2 作成DDL

```sql
-- ============================================================
-- 取引番号採番シーケンス
-- 用途: bank_account MASTERテーブルの torihiki_no 自動採番
-- フォーマット: 'BK' || LPAD(nextval('torihiki_seq')::text, 6, '0')
-- 採番例: BK000001, BK000002, BK000003, ...
-- ============================================================
CREATE SEQUENCE IF NOT EXISTS torihiki_seq
    START WITH 1
    INCREMENT BY 1
    MINVALUE 1
    MAXVALUE 999999
    NO CYCLE;
```

### 2.3 採番フォーマット

| 構成要素 | 値 | 説明 |
|---------|-----|------|
| 固定プレフィックス | `BK` | Bank の略称。銀行口座 MASTER であることを示す |
| 数値部 | 6桁ゼロ埋め | `LPAD(nextval('torihiki_seq')::text, 6, '0')` |
| フォーマット全体 | `BKxxxxxx` | 例: BK000001, BK000002, ... BK999999 |

**採番例**:

| nextval 戻り値 | フォーマット後 |
|:------------:|:------------:|
| 1 | BK000001 |
| 2 | BK000002 |
| 10 | BK000010 |
| 100 | BK000100 |
| 1000 | BK001000 |
| 999999 | BK999999（最終値） |

### 2.4 デフォルト値設定

bank_account テーブルの torihiki_no カラムに DEFAULT として設定:

```sql
torihiki_no VARCHAR(20) NOT NULL UNIQUE
    DEFAULT 'BK' || LPAD(nextval('torihiki_seq')::text, 6, '0')
```

INSERT 時に torihiki_no を省略すると自動採番される:

```sql
-- 自動採番（torihiki_no 省略）
INSERT INTO bank_account (branch_code, bank_name, account_type, account_number, account_holder)
VALUES ('038', '三菱UFJ銀行', '普通', '7654321', 'カ）テクノトウキョウ');
-- → torihiki_no は 'BK000001' が自動設定

-- 明示指定も可能（特殊ケース用）
INSERT INTO bank_account (torihiki_no, branch_code, bank_name, account_type, account_number, account_holder)
VALUES ('BK000100', '001', '三井住友銀行', '当座', '1111111', 'カ）テスト');
```

### 2.5 運用ルール

| 項目 | ルール |
|------|--------|
| 採番タイミング | INSERT 時（DEFAULT 評価時） |
| 欠番発生条件 | INSERT 失敗（ロールバック）時 |
| 欠番の扱い | 欠番を詰めない（連番保証不要） |
| 手動設定 | 特殊ケースのみ許可（連番重複に注意） |
| 上限到達時 | エラー発生。MAXVALUE 拡張またはリセットで対応 |
| リセット方法 | `ALTER SEQUENCE torihiki_seq RESTART WITH 1;`（要管理者権限） |

### 2.6 関連テーブル

| テーブル | カラム | 関係 | 説明 |
|----------|--------|------|------|
| bank_account | torihiki_no | DEFAULT 使用 | MASTERテーブル。取引番号を自動採番 |
| customer | torihiki_no | FK 参照 | 業務テーブル。MASTERの取引番号を保存 |

## 3. シーケンス管理SQL

### 3.1 状態確認

```sql
-- シーケンス一覧
SELECT schemaname, sequencename, start_value, min_value, max_value, increment_by, cycle, last_value
FROM pg_sequences
WHERE sequencename = 'torihiki_seq';

-- 現在値と次値
SELECT currval('torihiki_seq') AS current_value;  -- 要: 当該セッションでnextval実行済
SELECT nextval('torihiki_seq') AS next_value;      -- 次の値を取得（進む）
```

### 3.2 メンテナンス

```sql
-- リセット（初期化時など）
ALTER SEQUENCE torihiki_seq RESTART WITH 1;

-- 特定値に設定
SELECT setval('torihiki_seq', 100);

-- 最大値拡張（データ移行時など）
ALTER SEQUENCE torihiki_seq MAXVALUE 9999999;

-- シーケンス削除（要: 依存解除）
-- DROP SEQUENCE IF EXISTS torihiki_seq CASCADE;
```
