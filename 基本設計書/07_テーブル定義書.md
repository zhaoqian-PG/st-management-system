## 1. user（ユーザー）

| 論理名 | 物理名 | データ型 | PK | NOT NULL | DEFAULT | 説明 |
|--------|--------|---------|:--:|:--------:|---------|------|
| ID | id | BIGSERIAL | ○ | ○ | 自動採番 | 主キー |
| ユーザー名 | username | VARCHAR(50) | - | ○ | - | ログインID（一意） |
| パスワード | password | VARCHAR(255) | - | ○ | - | BCryptハッシュ化 |
| ロール | role | VARCHAR(20) | - | ○ | 'USER' | ADMIN / USER |
| 作成日時 | create_time | TIMESTAMP | - | ○ | NOW() | レコード作成日時 |
| 更新日時 | update_time | TIMESTAMP | - | ○ | NOW() | レコード更新日時 |

**制約:** UNIQUE (username)

## 3. customer（顧客）★業務テーブル

銀行口座情報は bank_account MASTERテーブルで管理。customer は bank_account から FK 参照される業務テーブル。

| 論理名 | 物理名 | データ型 | PK | NOT NULL | DEFAULT | 説明 |
|--------|--------|---------|:--:|:--------:|---------|------|
| ID | id | BIGSERIAL | ○ | ○ | 自動採番 | 主キー |
| 顧客番号 | customer_code | VARCHAR(20) | - | ○ | 自動採番（CUS+4桁） | 顧客識別番号（一意） |
| 会社名 | company_name | VARCHAR(200) | - | ○ | - | 会社正式名称 |
| 社長名 | president_name | VARCHAR(100) | - | - | - | 代表取締役・社長氏名 |
| Webサイト | website | VARCHAR(500) | - | - | - | 会社WebサイトURL |
| 住所 | address | VARCHAR(500) | - | - | - | 会社住所 |
| 担当者名 | contact_name | VARCHAR(100) | - | - | - | 窓口担当者 |
| メール | email | VARCHAR(255) | - | - | - | 会社代表メール |
| 電話番号 | phone | VARCHAR(20) | - | - | - | 会社代表電話番号 |
| 営業担当 氏名 | sales_rep_name | VARCHAR(100) | - | - | - | 営業担当者氏名 |
| 営業担当 連絡先 | sales_rep_phone | VARCHAR(20) | - | - | - | 営業担当者電話番号 |
| 営業担当 メール | sales_rep_email | VARCHAR(255) | - | - | - | 営業担当者メールアドレス |
| 事務担当 氏名 | admin_rep_name | VARCHAR(100) | - | - | - | 事務担当者氏名 |
| 事務担当 連絡先 | admin_rep_phone | VARCHAR(20) | - | - | - | 事務担当者電話番号 |
| 事務担当 メール | admin_rep_email | VARCHAR(255) | - | - | - | 事務担当者メールアドレス |
| **取引番号** | **torihiki_no** | **VARCHAR(500)** | - | - | **NULL** | **紐付く取引番号（bank_accountと連携、複数時カンマ区切り）** |
| 作成日時 | create_time | TIMESTAMP | - | ○ | NOW() | レコード作成日時 |
| 更新日時 | update_time | TIMESTAMP | - | ○ | NOW() | レコード更新日時 |

**制約:** UNIQUE (customer_code)

## 4. bank_account（銀行口座）★MASTERテーブル

取引番号（torihiki_no）を自動採番。customer_id で顧客に紐付く（1:N）。

| 論理名 | 物理名 | データ型 | PK | NOT NULL | DEFAULT | 説明 |
|--------|--------|---------|:--:|:--------:|---------|------|
| ID | id | BIGSERIAL | ○ | ○ | 自動採番 | 内部主キー |
| 取引番号 | torihiki_no | VARCHAR(20) | - | ○ | BK+6桁自動採番 | 一意の取引番号（BK000001〜） |
| 顧客ID | customer_id | BIGINT | - | - | NULL | FK → customer(id) / 1顧客:多口座 |
| 支店番号 | branch_code | VARCHAR(10) | - | - | - | 銀行支店番号 |
| 銀行名称 | bank_name | VARCHAR(200) | - | ○ | - | 銀行名 |
| 口座種類 | account_type | VARCHAR(20) | - | ○ | - | 普通 / 当座 |
| 口座番号 | account_number | VARCHAR(50) | - | ○ | - | 口座番号 |
| 口座名義 | account_holder | VARCHAR(100) | - | ○ | - | 口座名義人 |
| 作成日時 | create_time | TIMESTAMP | - | ○ | NOW() | レコード作成日時 |
| 更新日時 | update_time | TIMESTAMP | - | ○ | NOW() | レコード更新日時 |

**シーケンス:** torihiki_seq START 1, フォーマット `'BK' || LPAD(nextval, 6, '0')`  
**制約:** UNIQUE (torihiki_no), FK (customer_id) → customer(id) ON DELETE SET NULL, CHECK (account_type IN ('普通', '当座'))  
**インデックス:** idx_ba_torihiki (torihiki_no) UNIQUE, idx_ba_customer (customer_id)

## 6. invoice（請求書）

| 論理名 | 物理名 | データ型 | PK | NOT NULL | DEFAULT | 説明 |
|--------|--------|---------|:--:|:--------:|---------|------|
| ID | id | BIGSERIAL | ○ | ○ | 自動採番 | 主キー |
| 請求書番号 | invoice_number | VARCHAR(50) | - | ○ | - | 請求書識別番号（一意） |
| 顧客ID | customer_id | BIGINT | - | ○ | - | FK → customer(id) |
| 年度 | year | INT | - | ○ | - | 請求対象年 |
| 月度 | month | INT | - | ○ | - | 請求対象月 (1-12) |
| 請求日 | invoice_date | DATE | - | - | - | 請求日付 |
| 支払期限 | due_date | DATE | - | - | - | 支払期限日 |
| 件名 | subject | VARCHAR(500) | - | - | - | 件名・プロジェクト名 |
| 税抜金額 | amount | DECIMAL(12,2) | - | ○ | 0 | 請求金額（税抜） |
| 消費税率 | tax_rate | DECIMAL(4,2) | - | - | 10.00 | 消費税率（%） |
| 消費税額 | tax_amount | DECIMAL(12,2) | - | - | 0 | 消費税額（自動計算） |
| 税込合計 | total_with_tax | DECIMAL(12,2) | - | - | 0 | 税込合計金額（自動計算） |
| ステータス | status | VARCHAR(20) | - | ○ | '下書き' | 下書き / 送付済 / 入金済 |
| 備考 | remark | TEXT | - | - | - | 備考・摘要 |
| 作成日時 | create_time | TIMESTAMP | - | ○ | NOW() | レコード作成日時 |
| 更新日時 | update_time | TIMESTAMP | - | ○ | NOW() | レコード更新日時 |

**制約:** UNIQUE (invoice_number), FK (customer_id) → customer(id) ON DELETE CASCADE, CHECK (month BETWEEN 1 AND 12)  
**インデックス:** idx_invoice_customer_id, idx_invoice_year_month (year, month)

## 7. invoice_detail（請求明細）

| 論理名 | 物理名 | データ型 | PK | NOT NULL | DEFAULT | 説明 |
|--------|--------|---------|:--:|:--------:|---------|------|
| ID | id | BIGSERIAL | ○ | ○ | 自動採番 | 主キー |
| 請求書ID | invoice_id | BIGINT | - | ○ | - | FK → invoice(id) ON DELETE CASCADE |
| 社員ID | employee_id | BIGINT | - | - | NULL | FK → employee(id)（正社員時） |
| 担当者名 | employee_name | VARCHAR(100) | - | - | - | 担当者名（BP/個人事業主対応） |
| 項目名 | description | VARCHAR(500) | - | - | - | 作業内容・項目名 |
| 数量 | quantity | DECIMAL(8,2) | - | ○ | 1 | 作業時間など |
| 単価 | unit_price | DECIMAL(12,2) | - | ○ | 0 | 時間単価など |
| 金額 | amount | DECIMAL(12,2) | - | ○ | 0 | quantity × unit_price |
| 残業 | is_overtime | BOOLEAN | - | ○ | FALSE | 残業フラグ |
| 作成日時 | create_time | TIMESTAMP | - | ○ | NOW() | レコード作成日時 |

**制約:** FK (invoice_id) → invoice(id) ON DELETE CASCADE, FK (employee_id) → employee(id) ON DELETE SET NULL  
**インデックス:** idx_inv_det_invoice (invoice_id)

## 8. FK関係図

```
│ employee │──1:N──│attendance│  ON DELETE CASCADE
└──────────┘       └──────────┘

┌──────────┐       ┌──────────────┐
│ customer │──1:N──│ bank_account │  ON DELETE SET NULL
│ (業務)    │◄──────│ (MASTER)      │  FK: customer_id
└────┬─────┘       └──────────────┘
     │
     │ 1:N
     ▼
┌──────────┐       ┌────────────────┐
│ invoice  │──1:N──│order_documents │  ON DELETE CASCADE
└──────────┘       └────────────────┘
```
