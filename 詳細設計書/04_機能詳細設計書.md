## 2. 認証機能 (Auth)

### 2.1 F-AUTH-01: ログイン

| 項目 | 内容 |
|------|------|
| 機能ID | F-AUTH-01 |
| 画面 | SCR-LOGIN |
| 操作種別 | C (Create) |
| API | POST /api/auth/login |

**処理フロー**:

```
[開始]
  │
  ├─ 1. ユーザーがユーザー名・パスワードを入力
  │
  ├─ 2. フロントエンド: フォームバリデーション
  │    ├─ ユーザー名 空チェック
  │    └─ パスワード 空チェック
  │    → バリデーションエラー時: インラインエラー表示 → [終了]
  │
  ├─ 3. フロントエンド: POST /api/auth/login {username, password}
  │
  ├─ 4. バックエンド AuthController.login():
  │    │
  │    ├─ 4.1 LoginRequest 受信
  │    │
  │    ├─ 4.2 AuthService.authenticate():
  │    │    ├─ UserRepository.findByUsername(username)
  │    │    │   → ユーザー不在: throw BadRequestException("認証失敗")
  │    │    ├─ BCryptPasswordEncoder.matches(password, user.password)
  │    │    │   → パスワード不一致: throw BadRequestException("認証失敗")
  │    │    └─ JwtTokenProvider.generateToken(username, role)
  │    │        → JWT文字列を返却
  │    │
  │    └─ 4.3 LoginResponse {token, type:"Bearer", username, role} 返却
  │
  ├─ 5. フロントエンド: レスポンス受信
  │    ├─ localStorage.setItem("token", token)
  │    ├─ setUser({username, role})
  │    └─ navigate("/employee")
  │
  └─ [終了: 認証成功]
```

**セキュリティ上の配慮**:
- 認証失敗時のメッセージは「ユーザー名またはパスワードが間違っています」に統一（ユーザー名の存在有無を推測させない）
- パスワードの連続失敗に対するアカウントロックは Ver 1.0 では未実装

### 3.1 F-EMP-01: 社員一覧表示

| 項目 | 内容 |
|------|------|
| 機能ID | F-EMP-01 |
| 画面 | SCR-EMP |
| 操作種別 | R (Read) |
| API | GET /api/employee |

**リクエストパラメータ**:

| パラメータ | 型 | 必須 | 説明 |
|-----------|------|:--:|------|
| keyword | String | - | 社員番号/氏名 部分一致 (LIKE %keyword%) |
| department | String | - | 部署フィルター |
| page | int | - | ページ番号 (0-based, default=0) |
| size | int | - | ページサイズ (default=10) |

**レスポンス**:

```json
{
  "data": {
    "content": [ EmployeeDTO, ... ],
    "totalElements": 5,
    "totalPages": 1,
    "number": 0,
    "size": 10,
    "first": true,
    "last": true
  }
}
```

**Service 層 実装ロジック**:

```java
public Page<EmployeeDTO> getEmployees(String keyword, String department, Pageable pageable) {
    Specification<Employee> spec = Specification.where(null);

    if (keyword != null && !keyword.isBlank()) {
        spec = spec.and((root, query, cb) -> cb.or(
            cb.like(root.get("employeeCode"), "%" + keyword + "%"),
            cb.like(root.get("name"), "%" + keyword + "%")
        ));
    }
    if (department != null && !department.isBlank()) {
        spec = spec.and((root, query, cb) ->
            cb.equal(root.get("department"), department));
    }

    Page<Employee> page = employeeRepository.findAll(spec, pageable);
    return page.map(this::convertToDTO);
}
```

### 3.3 F-EMP-03: 社員新規登録

| 項目 | 内容 |
|------|------|
| 機能ID | F-EMP-03 |
| 操作種別 | C (Create) |
| API | POST /api/employee |

**バリデーションルール**:

| フィールド | ルール | エラーメッセージ |
|-----------|--------|-----------------|
| employeeCode | @NotBlank, 最大20文字, 一意 | 社員番号は必須です / この社員番号は既に使用されています |
| name | @NotBlank, 最大100文字 | 氏名は必須です |
| email | @Email (任意), 最大255文字 | メールアドレスの形式が正しくありません |

**Service 層 実装ロジック**:

```java
@Transactional
public EmployeeDTO createEmployee(EmployeeDTO dto) {
    // 1. employee_code 一意チェック
    if (employeeRepository.findByEmployeeCode(dto.getEmployeeCode()).isPresent()) {
        throw new BadRequestException("この社員番号は既に使用されています");
    }

    // 2. DTO → Entity 変換
    Employee employee = new Employee();
    BeanUtils.copyProperties(dto, employee);
    employee.setCreateTime(LocalDateTime.now());
    employee.setUpdateTime(LocalDateTime.now());

    // 3. 保存
    employee = employeeRepository.save(employee);

    // 4. Entity → DTO 変換して返却
    return convertToDTO(employee);
}
```

**HTTPステータス**:
- 成功: 201 Created
- バリデーションエラー: 400 Bad Request
- 一意制約違反: 400 Bad Request

**バックエンド処理**:

```java
@Transactional
public void deleteEmployee(Long id) {
    Employee employee = employeeRepository.findById(id)
        .orElseThrow(() -> new ResourceNotFoundException("社員が見つかりません: " + id));

    // 添付ファイルの物理削除
    if (employee.getAttachmentPath() != null) {
        fileStorageService.delete(employee.getAttachmentPath());
    }

    // 社員に紐付く勤務記録の削除 (CASCADE または個別削除)
    attendanceRepository.deleteByEmployeeId(id);

    // 社員レコードの削除
    employeeRepository.delete(employee);
}
```

**削除ポリシー**: 物理削除。紐付く勤務記録も全て削除する。

```java
public ResponseEntity<Resource> downloadFile(Long employeeId) {
    Employee employee = employeeRepository.findById(employeeId)
        .orElseThrow(() -> new ResourceNotFoundException("社員が見つかりません"));

    if (employee.getAttachmentPath() == null) {
        throw new ResourceNotFoundException("添付ファイルがありません");
    }

    Path filePath = fileStorageService.load(employee.getAttachmentPath());
    Resource resource = new UrlResource(filePath.toUri());

    if (!resource.exists()) {
        throw new ResourceNotFoundException("ファイルが見つかりません");
    }

    return ResponseEntity.ok()
        .contentType(MediaType.APPLICATION_OCTET_STREAM)
        .header(HttpHeaders.CONTENT_DISPOSITION,
            "attachment; filename=\"" + resource.getFilename() + "\"")
        .body(resource);
}
```

## 5. 勤務管理機能 (Attendance)

### 5.1 F-ATT-01: 勤務一覧表示 / F-ATT-02: 勤務絞込

| 項目 | 内容 |
|------|------|
| 機能ID | F-ATT-01, F-ATT-02 |
| API | GET /api/attendance |

**リクエストパラメータ**:

| パラメータ | 型 | 必須 | 説明 |
|-----------|------|:--:|------|
| year | int | - | 年度フィルター |
| month | int | - | 月度フィルター (1-12) |
| employeeId | Long | - | 社員IDフィルター (省略=全社員) |
| page | int | - | ページ番号 |
| size | int | - | ページサイズ |

**Service層 クエリビルド**:

```java
public Page<AttendanceDTO> getAttendances(Integer year, Integer month,
        Long employeeId, Pageable pageable) {
    Specification<Attendance> spec = Specification.where(null);

    if (employeeId != null) {
        spec = spec.and((root, query, cb) ->
            cb.equal(root.get("employeeId"), employeeId));
    }
    if (year != null && month != null) {
        LocalDate start = LocalDate.of(year, month, 1);
        LocalDate end = start.plusMonths(1);
        spec = spec.and((root, query, cb) ->
            cb.between(root.get("workDate"), start, end));
    } else if (year != null) {
        LocalDate start = LocalDate.of(year, 1, 1);
        LocalDate end = LocalDate.of(year, 12, 31);
        spec = spec.and((root, query, cb) ->
            cb.between(root.get("workDate"), start, end));
    }

    Page<Attendance> page = attendanceRepository.findAll(spec, pageable);
    return page.map(entity -> {
        AttendanceDTO dto = convertToDTO(entity);
        // JOIN で社員名を取得
        employeeRepository.findById(entity.getEmployeeId())
            .ifPresent(emp -> dto.setEmployeeName(emp.getName()));
        return dto;
    });
}
```

### 5.3 F-ATT-06: CSVエクスポート

| 項目 | 内容 |
|------|------|
| API | GET /api/attendance/export |

**処理フロー**:

```java
public void exportCsv(HttpServletResponse response, Integer year,
        Integer month, Long employeeId) throws IOException {
    // 1. ページネーションなしで全件取得
    List<Attendance> list = attendanceRepository.findAll(
        buildSpec(year, month, employeeId));

    // 2. Entity → DTO + 社員名付与
    List<AttendanceDTO> dtos = list.stream()
        .map(entity -> {
            AttendanceDTO dto = convertToDTO(entity);
            employeeRepository.findById(entity.getEmployeeId())
                .ifPresent(emp -> dto.setEmployeeName(emp.getName()));
            return dto;
        })
        .sorted(Comparator.comparing(AttendanceDTO::getWorkDate)
            .thenComparing(AttendanceDTO::getEmployeeName))
        .collect(Collectors.toList());

    // 3. CSV出力 (BOM付きUTF-8)
    CsvExportUtil.writeCsv(response, dtos, year, month);
}
```

## 7. エラーハンドリング統一仕様

### 7.1 エラー分類とHTTPステータス

| エラー種別 | HTTPステータス | 例外クラス |
