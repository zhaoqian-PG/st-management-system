## 2. 認証 (Authentication)

### 2.1 JWT トークン設計

**トークン発行フロー**:

```
[クライアント]                    [サーバー]
     │                                │
     │  POST /api/auth/login          │
     │  {username, password}          │
     │ ─────────────────────────────> │
     │                                ├─ 1. UserRepository からユーザー検索
     │                                ├─ 2. BCryptPasswordEncoder.matches()
     │                                ├─ 3. JWT 生成 (HMAC-SHA256)
     │                                │     payload: {sub, role, iat, exp}
     │                                │     署名: secret key でHMAC
     │  {token, type, username, role} │
     │ <───────────────────────────── │
     │                                │
     │  localStorage.setItem(token)   │
     │                                │
     │  (以降の全リクエスト)           │
     │  Authorization: Bearer {token} │
     │ ─────────────────────────────> │
     │                                ├─ JwtAuthFilter:
     │                                │   1. ヘッダーからトークン抽出
     │                                │   2. 署名検証
     │                                │   3. 有効期限チェック
     │                                │   4. SecurityContextHolder に認証情報設定
     │                                │
```

### 2.2 JWT設定パラメータ

| パラメータ | 値 | 説明 |
|-----------|------|------|
| アルゴリズム | HMAC-SHA256 (HS256) | 署名アルゴリズム |
| 秘密鍵 | 環境変数 `JWT_SECRET` (デフォルト: `st-management-secret-key-2026`) | 256bit以上のランダム文字列推奨 |
| 有効期限 | 24時間 (86400000ms) | 本番向けには8時間以内を推奨 |
| 発行者 (iss) | st-management | (未使用・将来対応) |

### 2.3 トークン更新戦略

| 手法 | Ver 1.0 | Ver 2.0予定 |
|------|:------:|:----------:|
| リフレッシュトークン | なし（シングルトークン方式） | 検討 |
| トークン期限切れ時 | ログイン画面へリダイレクト | リフレッシュトークンで透過更新 |
| スライディングセッション | なし | 検討 |

### 2.4 JwtTokenProvider 実装詳細

```java
@Component
public class JwtTokenProvider {

    @Value("${app.jwt.secret}")
    private String secret;

    @Value("${app.jwt.expiration-ms}")
    private long expirationMs;

    @PostConstruct
    protected void init() {
        // 秘密鍵をBase64エンコードしてHS256で使用可能な形式に変換
        secret = Base64.getEncoder().encodeToString(secret.getBytes(StandardCharsets.UTF_8));
    }

    private SecretKey getSigningKey() {
        byte[] keyBytes = Decoders.BASE64.decode(secret);
        return Keys.hmacShaKeyFor(keyBytes);
    }

    public String generateToken(String username, String role) {
        Date now = new Date();
        Date expiryDate = new Date(now.getTime() + expirationMs);

        return Jwts.builder()
                .subject(username)
                .claim("role", role)
                .issuedAt(now)
                .expiration(expiryDate)
                .signWith(getSigningKey())
                .compact();
    }

    public String getUsernameFromToken(String token) {
        Claims claims = Jwts.parser()
                .verifyWith(getSigningKey())
                .build()
                .parseSignedClaims(token)
                .getPayload();
        return claims.getSubject();
    }

    public String getRoleFromToken(String token) {
        Claims claims = Jwts.parser()
                .verifyWith(getSigningKey())
                .build()
                .parseSignedClaims(token)
                .getPayload();
        return claims.get("role", String.class);
    }

    public boolean validateToken(String token) {
        try {
            Jwts.parser()
                .verifyWith(getSigningKey())
                .build()
                .parseSignedClaims(token);
            return true;
        } catch (JwtException | IllegalArgumentException e) {
            // トークン期限切れ、署名不正、形式不正 など
            return false;
        }
    }
}
```

### 2.5 JwtAuthFilter 実装詳細

```java
@Component
public class JwtAuthFilter extends OncePerRequestFilter {

    @Autowired
    private JwtTokenProvider tokenProvider;

    @Autowired
    private UserDetailsServiceImpl userDetailsService;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
            HttpServletResponse response, FilterChain filterChain)
            throws ServletException, IOException {

        // 1. Authorization ヘッダーからトークン抽出
        String token = extractToken(request);

        // 2. トークン検証
        if (token != null && tokenProvider.validateToken(token)) {
            String username = tokenProvider.getUsernameFromToken(token);
            String role = tokenProvider.getRoleFromToken(token);

            // 3. UserDetails をロードし、認証オブジェクトを構築
            UserDetails userDetails = userDetailsService.loadUserByUsername(username);

            List<SimpleGrantedAuthority> authorities =
                Collections.singletonList(new SimpleGrantedAuthority("ROLE_" + role));

            UsernamePasswordAuthenticationToken authentication =
                new UsernamePasswordAuthenticationToken(
                    userDetails, null, authorities);

            // 4. SecurityContext に設定
            SecurityContextHolder.getContext().setAuthentication(authentication);
        }

        filterChain.doFilter(request, response);
    }

    private String extractToken(HttpServletRequest request) {
        String bearerToken = request.getHeader("Authorization");
        if (bearerToken != null && bearerToken.startsWith("Bearer ")) {
            return bearerToken.substring(7);
        }
        return null;
    }
}
```

### 3.2 SecurityConfig 設定

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Autowired
    private JwtAuthFilter jwtAuthFilter;

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            // CSRF: APIがステートレスなため無効化
            .csrf(csrf -> csrf.disable())

            // セッション: ステートレス (JWT管理)
            .sessionManagement(session ->
                session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))

            // エンドポイント権限設定
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/login").permitAll()
                .requestMatchers(HttpMethod.OPTIONS, "/**").permitAll()
                .requestMatchers("/api/**").authenticated()
                .anyRequest().permitAll()
            )

            // JWTフィルターをSpring Securityフィルターチェーンに追加
            .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class);

        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }

    @Bean
    public AuthenticationManager authenticationManager(
            AuthenticationConfiguration config) throws Exception {
        return config.getAuthenticationManager();
    }
}
```

## 6. ファイルアップロードセキュリティ

### 6.1 検証フロー

```
[ファイルアップロード受信]
  │
  ├─ 1. 拡張子検証
  │    禁止: .exe, .bat, .sh, .jar, .war, .php, .jsp, .html
  │    許可: .pdf, .doc, .docx, .xls, .xlsx, .jpg, .jpeg, .png
  │    → 不許可: 400 Bad Request
  │
  ├─ 2. ファイルサイズ検証
  │    max: 10MB = 10485760 bytes
  │    → 超過: 400 Bad Request
  │
  ├─ 3. MIMEタイプ検証 (追加推奨)
  │    Content-Type と拡張子の整合性を簡易チェック
  │
  ├─ 4. ファイル名のサニタイズ
  │    元ファイル名を保持しつつ、保存名はUUIDベース
  │    → パストラバーサル対策: "../" "..\\" を含む場合は拒否
  │
  ├─ 5. サーバー側保存
  │    ./uploads/{type}/{uuid}.{sanitized_ext}
  │    (WEB-INF外、直接アクセス不可領域に保存)
  │
  └─ 6. DBにレコード登録
```

### 6.2 実装コード

```java
@Service
public class FileStorageService {

    private static final long MAX_FILE_SIZE = 10 * 1024 * 1024; // 10MB
    private static final Set<String> ALLOWED_EXTENSIONS = Set.of(
        ".pdf", ".doc", ".docx", ".xls", ".xlsx", ".jpg", ".jpeg", ".png"
    );

    @Value("${app.upload.dir}")
    private String uploadDir;

    public String store(MultipartFile file, String subDir) throws IOException {
        // 1. 拡張子検証
        String originalName = file.getOriginalFilename();
        if (originalName == null || originalName.isBlank()) {
            throw new BadRequestException("ファイル名が空です");
        }

        // パストラバーサルチェック
        if (originalName.contains("..") || originalName.contains("/") || originalName.contains("\\")) {
            throw new BadRequestException("不正なファイル名です");
        }

        String ext = originalName.substring(originalName.lastIndexOf('.')).toLowerCase();
        if (!ALLOWED_EXTENSIONS.contains(ext)) {
            throw new BadRequestException("サポートされていないファイル形式です: " + ext);
        }

        // 2. サイズ検証
        if (file.getSize() > MAX_FILE_SIZE) {
            throw new BadRequestException("ファイルサイズが制限を超えています (最大10MB)");
        }

        // 3. 保存
        String storedName = UUID.randomUUID().toString() + ext;
        Path dir = Paths.get(uploadDir, subDir);
        Files.createDirectories(dir);
        Path target = dir.resolve(storedName);
        Files.copy(file.getInputStream(), target, StandardCopyOption.REPLACE_EXISTING);

        return subDir + "/" + storedName;
    }

    public Path load(String filePath) {
        return Paths.get(uploadDir).resolve(filePath).normalize();
    }

    public void delete(String filePath) {
        try {
            Files.deleteIfExists(load(filePath));
        } catch (IOException e) {
            throw new RuntimeException("ファイルの削除に失敗しました", e);
        }
    }
}
```

## 7. XSS 対策

### 7.1 フロントエンド

- React の JSX はデフォルトで全文字列をエスケープするため、通常のデータ表示ではXSS脆弱性は発生しない
- `dangerouslySetInnerHTML` は使用禁止（コードレビューで強制）

### 7.2 バックエンド

```java
// 入力値は必要に応じてサニタイズ (HTMLタグをエスケープ)
public static String sanitize(String input) {
    if (input == null) return null;
    return input
        .replace("&", "&amp;")
        .replace("<", "&lt;")
        .replace(">", "&gt;")
        .replace("\"", "&quot;")
        .replace("'", "&#x27;");
}
```

## 8. SQLインジェクション対策

- Spring Data JPA は全クエリで PreparedStatement を使用するため、基本的にSQLインジェクション脆弱性は発生しない
- 動的クエリには `Specification` + `CriteriaBuilder` を使用し、文字列連結を避ける
- `@Query` アノテーションでのネイティブSQL使用時は必ずパラメータバインディング (`:param` 構文) を使用する

## 10. セキュリティチェックリスト

| # | チェック項目 | 対応状況 |
|:--|------------|:------:|
| 1 | JWT秘密鍵は環境変数で管理されているか | ○ `JWT_SECRET` |
| 2 | パスワードはBCryptでハッシュ化されているか | ○ strength=10 |
| 3 | ログイン失敗時のエラーメッセージは曖昧か | ○ ユーザー名の存在を推測不可 |
| 4 | CORSは信頼できるオリジンのみ許可しているか | ○ localhost:3000限定 |
| 5 | ファイルアップロードの拡張子が制限されているか | ○ ホワイトリスト方式 |
| 6 | ファイルアップロードのサイズが制限されているか | ○ 10MB |
| 7 | パストラバーサル対策があるか | ○ ファイル名サニタイズ + UUID保存 |
| 8 | SQLインジェクション対策があるか | ○ JPA + CriteriaBuilder |
| 9 | XSS対策があるか | ○ Reactデフォルトエスケープ |
| 10 | エラーレスポンスに機密情報を含めていないか | ○ 統一エラーフォーマット |
| 11 | パスワードがログに出力されていないか | ○ toString()除外 |
| 12 | CSRF対策があるか | ○ APIがステートレスなためJWTで代替 |
| 13 | ログアウト時にクライアント側トークンが破棄されるか | ○ localStorage.removeItem |
| 14 | トークンに有効期限が設定されているか | ○ 24時間 |
| 15 | 認証不要APIが適切に制限されているか | ○ /api/auth/login のみ |
